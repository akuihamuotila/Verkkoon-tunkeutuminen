Karvinen 2025: Network Interface Names on Linux:

- Verkkoliitäntä on kuin verkkokortti, mutta se voi olla myös virtuaalinen. Esimerkiksi lo (loopback) ei vastaa mitään fyysistä korttia

- Systemd-pohjaisissa Linux-distroissa (kuten esim. Debian) liitännät nimetään etuliitteen mukaan: en = langallinen Ethernet, wl = WiFi, lo = loopback

- Numerot etuliitteen jälkeen (esim. enp1s0) kertovat laitteen sijainnin väylällä tai sen MAC-osoitteen. Tämä takaa, että sama nimi viittaa aina samaan korttiin uudelleenkäynnistysten välillä

## d)

Otetaan edellisen kohdan kaappauksesta seuraava paketti`4687`

```bash
4687	31.514074151	10.0.2.15	142.251.38.78	TCP	54	56862 → 443 [FIN, ACK] Seq=2101 Ack=8334 Win=65535 Len=0
```

`4687` paketin järjestysnumero kaapauksessa

`31.514074151` aikaleima sekunteina kaappauksen alusta

`10.0.2.15` lähettäjän IP-osoite, eli Kali-virtuaalikoneeni

`142.251.38.78` vastaanottajan IP-osoite, eli Googlen palvelin, koska menin sivulle `google.com`

`TCP` protokolla

`54` paketin koko tavuina

`56862 → 443` porttinumerot, 56862 on selaimeni käyttämä satunnainen lähdeportti, 443 on HTTPS:n vakioportti palvelimen päässä

`[FIN, ACK]` TCP-liput, FIN tarkoittaa että lähettäjä haluaa sulkea yhteyden, ACK kuittaa aiemmin vastaanotetut paketit

`Seq=2101` järjestysnumero, kertoo missä kohtaa datavirtaa tämä paketti on

`Ack=8334` kuittausnumero, kertoo minkä tavun odottaa seuraavaksi palvelimelta

`Win=65535` vastaanottopuskurin koko tavuina, kertoo palvelimelle kuinka paljon dataa voidaan lähettää kerralla

`Len=0` paketin hyötykuorma on tyhjä, eli tämä on pelkkä ohjausviesti ilman dataa

```bash
4687	31.514074151	10.0.2.15	142.251.38.78	TCP	54	56862 → 443 [FIN, ACK] Seq=2101 Ack=8334 Win=65535 Len=0
```

**1. Linkkikerros (Link Layer)** Paketti kulkee virtuaaliverkon Ethernet-kehyksessä lähiverkossa MAC-osoitteiden avulla

**2. Verkkokerros (Internet Layer)** -- IP: `10.0.2.15 → 142.251.38.78`

**3. Kuljetuskerros (Transport Layer)** -- TCP: `56862 → 443 [FIN, ACK] Seq=2101 Ack=8334 Win=65535 Len=0`

**4. Sovelluskerros (Application Layer)** -- HTTPS (TLS): portti 443 kertoo että kyseessä on salattu HTTPS-yhteys

## h)

Kirjoitan wiresharkissa display filter -kenttään `tls.handshake.type == 1` ja painoin ENTER

TLS-handshake on prosessi, jossa selain ja palvelin sopivat salausasetuksista ennen kuin varsinainen salattu liikenne alkaa.

Handshake koostuu useista eri viesteistä joilla on omat numerokoodinsa:

`1` = Client Hello

`2` = Server Hello

`11` = Certificate

`20` = Finished

Filter `tls.handshake.type == 1` näyttää siis vain Client Hello -viestit, koska juuri siinä näkyy SNI (Server Name Indication) -kenttä selkokielisenä.

Se on ainoa kohta koko TLS-yhteydessä jossa palvelimen nimi näkyy salaamattomana, vaikka itse liikenne on salattu.

**Paketit 1-2: DNS-kysely**

Ennen kuin ping voi lähettää yhtään pakettia, koneen täytyy selvittää `terokarvinen.com`-nimeä vastaava IP-osoite.

Kone lähettää kaksi DNS-kyselyä DNS-palvelimelle `10.0.2.3` (VirtualBoxin sisäinen reititin):

Paketti 2 kyselee IPv6-osoitetta samanaikaisesti (AAAA-tietue, 128-bittinen osoite, neljä kertaa pidempi kuin A-tietue)

**Paketit 3-4: DNS-vastaus**

DNS-palvelin vastaa:

Paketti 3: IPv4-osoite on `139.162.131.217` tämä on palvelin jolla `terokarvinen.com` sijaitsee

Paketti 4: IPv6-osoitetta ei ole, palautetaan SOA-tietue (Start of Authority) joka kertoo kuka hallinnoi domainia, tässä tapauksessa `ns1.gandi.net`

**Paketit 5-6: ICMP ping**

Nyt kun IP-osoite on tiedossa, ping voi lähteä:

Paketti 5: kone lähettää ICMP Echo Request -paketin (`ping request`) osoitteeseen `139.162.131.217`, TTL=64

Paketti 6: palvelin vastaa ICMP Echo Reply -paketilla (`ping reply`), TTL=63 -- TTL on pienentynyt yhdellä koska paketti on kulkenut yhden reitittimen läpi

Pakettien 5 ja 6 aikaleimasta näkee verkon vasteajan: `0.126736953 - 0.086066566 = 0.040670387` sekuntia eli noin 40.7 ms, joka täsmää ping-komennon tulostaman ajan kanssa.

**Paketit 7-8: käänteinen DNS-kysely (PTR)**

Ping tekee automaattisesti käänteisen DNS-kyselyn. Se selvittää mitä nimeä IP-osoite `139.162.131.217` vastaa:

Paketti 7: kysely muodossa `217.131.162.139.in-addr.arpa` (IP-osoite käännettynä)

Paketti 8: vastaus on `li1397-217.members.linode.com` tämä näkyi myös ping-komennon tulosteessa
